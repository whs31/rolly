rolly::box class

Owning unique smart pointer which can never be null.

Template parameters
T Type of data stored in the box. Must fulfill std::destructible requirement.

The box class behaves very similarly to the std::unique_ptr class, but unlike the std::unique_ptr, it is 100% guaranteed to be filled at all times (unless the user tries to access it after calling std::move does something else that is clearly a bad idea).

Reference implementation is taken from the reflect-cpp library.

API reference matches (almost) the Rust Language implementation.

See also

https://github.com/getml/reflect-cpp/blob/main/include/rfl/Box.hpp

https://doc.rust-lang.org/std/boxed/struct.Box.html

templateÂ <typename T>
T* rolly::box::leak() noexcept

Consumes and leaks the box, returning a mutable pointer to the underlying object.

Returns Mutable unmanaged pointer to the underlying object or nullptr if the box is already consumed.

The caller is responsible for freeing the memory by calling deleter on the returned pointer.

Note

Calling this function leaves this box in an invalid state. Trying to use it afterwards will result in contract violation.
